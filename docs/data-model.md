# Data Model

The database relies on a relational Postgres schema managed by Drizzle ORM.

## Core Entities

### Users (`users`)
Stores user identities and profile details.
- `id` (Primary Key)
- `email` (Unique, acts as the primary link for OAuth flows)
- `username` (Unique)
- `displayName`
- `passwordHash` (Required even for OAuth users, initialized randomly)
- `isAdmin`

### Posts (`posts`)
Represents the main threads of discussion.
- `id` (Primary Key)
- `authorId` (Foreign Key -> `users.id`)
- `section` (Enum: 'carpool', 'academic', 'roommate', 'other')
- `title`
- `body`
- `isAnonymous` (Boolean toggle for anonymous viewing)
- `authorNameSnapshot` (Cached author name at creation, critical for persistent anonymity)

### Comments (`comments`)
Threaded replies linked to posts.
- `id` (Primary Key)
- `postId` (Foreign Key -> `posts.id`)
- `authorId` (Foreign Key -> `users.id`)
- `parentId` (Self-referential Foreign Key -> `comments.id` for nesting)
- `body`
- `isAnonymous`
- `authorNameSnapshot`

### Bookmarks (`bookmarks`)
A junction table tracking which users have bookmarked which posts.
- `id` (Primary Key)
- `userId` (Foreign Key -> `users.id`)
- `postId` (Foreign Key -> `posts.id`)

### Notifications (`notifications`)
Tracks alerts generated by system interactions (e.g. replies).
- `id` (Primary Key)
- `userId` (Foreign Key -> `users.id` - The recipient)
- `postId` (Foreign Key -> `posts.id`)
- `type` (e.g., 'reply_to_post', 'reply_to_comment')
- `actorName` (Name of the user who triggered the notification)
- `postTitle` (Contextual title for rendering)
- `isRead`

### Drafts (`drafts`)
Autosaved content to prevent data loss.
- `id` (Primary Key)
- `userId` (Foreign Key -> `users.id`)
- `postId` (Optional Foreign Key -> `posts.id`)
- `content` (JSON payload of the draft state)

## Integrity Rules
- Deleting a `post` triggers `ON DELETE CASCADE` for all related `comments`, `bookmarks`, `notifications`, and `drafts`.
- Both `posts` and `comments` support "anonymity mode", which relies heavily on the `authorNameSnapshot` being set accurately at insert time, rather than queried dynamically via `authorId` later.
